# 马尔可夫决策过程 (MDP)

# 核心要素：四元组 (S,A,P,R)(S, A, P, R)(S,A,P,R)

  • SSS (States): 状态集合。
  • AAA (Actions): 动作集合。智能体可以采取的操作。
  • PPP (Transition Probability): 状态转移概率。即在状态 sss 下采取动作 aaa，变成状态 s′s's′ 的概率 P(s′∣s,a)P(s' | s, a)P(s′∣s,a)。
  • RRR (Reward): 奖励函数。执行动作后获得的即时反馈。

具有马尔可夫性：未来只取决于当前状态，而与过去无关

# 部分可观测马尔可夫决策过程 (POMDP)

在现实世界中，MDP 通常太理想了，所以引入更加通用的 POMDP。

# 核心要素：在 MDP 基础上增加了 (Ω,O)(Ω, O)(Ω,O)

  • ΩΩΩ (Observations): 观察集合。智能体实际看到的东西。
  • OOO (Observation Function): 观察概率。即在状态 sss 下，看到观察值 ooo 的概率 P(o∣s)P(o | s)P(o∣s)。

假设你在森林里。

  • 真实状态 sss：老虎藏在草丛里（这是事实）。
  • 观察值 ooo：你看到草丛抖动了一下。

如果状态确实是 sss，你的感官会有多大概率捕捉到 ooo。P(o∣s)P(o|s)P(o∣s) 就在描述这种关联：

  • 如果老虎真的在草丛里（sss），草丛抖动的概率可能是 0.80.80.8（P(抖动∣老虎在)=0.8P(\text{抖动} | \text{老虎在}) = 0.8P(抖动∣老虎在)=0.8）。
  • 但也有 0.20.20.2 的概率，老虎潜伏得非常好，草丛一点没动（P(不抖动∣老虎在)=0.2P(\text{不抖动} | \text{老虎在}) = 0.2P(不抖动∣老虎在)=0.2）。

# Planning

范式做法代表工作
RL as External GuideLLM 只采样动作序列（如 MCTS 等结构化采样），通过 RL 的方式（如 Q-Learning）训练 critic 网络评估并引导采样过程RAP、LATS、Learning-to-Plan、MAPF-DT、Planning without Search
RL as Internal Driver通过 RL 的方式直接优化 LLM 生成动作序列的能力Voyager、DSP、AdaPlan、Planner-R1

我们注意如果采用 MCTS 介入：它在每个关键决策点（比如：方案 A 用 asyncio，方案 B 用多线程），像下围棋（AlphaGo）一样，在脑海里向前模拟推演好几步，生成很多条不同的 "动作序列"（Trajectory / 思路分支）。而 Critic 就是根据 MCTS 当前生出的那个思维节点（State），吐出一个分数（Q 值）。通过剪枝优化（具体算法比较复杂），critic 和 MCTS 能力越强，搜索最优路径能力增强。但是注意这部分 LLM 能力是没有提升的，是外部的 plan 能力得到提升

Link

# ASPO — 尽早调用工具

在鼓励模型尽早调用工具的过程中会有问题，那就是：

Ai=Ri−mean(R)std(R)=ri′−mean(r′)std(r′)A_i = \frac{R_i - \text{mean}(\mathbf{R})}{\text{std}(\mathbf{R})} = \frac{r'_i - \text{mean}(\mathbf{r'})}{\text{std}(\mathbf{r'})}Ai​=std(R)Ri​−mean(R)​=std(r′)ri′​−mean(r′)​

全组模型都答对了题（基础分都是 1）。整个组内比拼的唯一标准变成了谁代码调用得早。如果一个样本答对了但是调用稍晚，它的 AiA_iAi​ 就会变成负值，在策略梯度更新时直接被反向惩罚。

所以对 Advantage 进行了修改：

Ai=Acorrect,i+clip(δ⋅pi−mean(p)mean(L),−k⋅Acorrect,i,k⋅Acorrect,i)A_i = A_{\text{correct}, i} + \text{clip}\left(\delta \cdot \frac{p_i - \text{mean}(\mathbf{p})}{\text{mean}(\mathbf{L})}, \; -k \cdot A_{\text{correct}, i}, \; k \cdot A_{\text{correct}, i}\right)Ai​=Acorrect,i​+clip(δ⋅mean(L)pi​−mean(p)​,−k⋅Acorrect,i​,k⋅Acorrect,i​)

前半部分不变，但是对于调用工具时间早晚的得分进行了约束，让得分的上下界限制在一个区间。

# OTC-PO — 调用工具别太频繁

这个 reward 得分类似 gated 得分，如果答错直接归 0，做对了再谈调用效率问题：

rϕtool(q,y)=α⋅rtool⋅rϕ(q,y)r_{\phi}^{tool}(q, y) = \alpha \cdot r_{tool} \cdot r_{\phi}(q, y)rϕtool​(q,y)=α⋅rtool​⋅rϕ​(q,y)

PPO 模式：

rtool=cos⁡(m⋅π2m+c)r_{tool} = \cos\left(\frac{m \cdot \pi}{2m + c}\right)rtool​=cos(2m+cm⋅π​)

因为还有 critic，所以在 PPO 中关注绝对分数，而 ccc 这个平滑常数：

  • ccc 越小：惩罚下降越快，表明对工具调用越严苛，极端鼓励少用工具。
  • ccc 越大：惩罚下降越慢，给模型的容错空间更大，更包容复杂的工具多轮交互。

GRPO 模式：

每个组内采样的轨迹得到一个估计调用次数的最佳数值（通常是一次 rollout 的最小值）：

f(m,n)={0,ifm=0andn=0m,ifn=02nmm+notherwisef(m,n) = \begin{cases} 0, & \text{if } m = 0 \text{ and } n = 0 \\ m, & \text{if } n = 0 \\ \dfrac{2nm}{m+n} & \text{otherwise} \end{cases}f(m,n)=⎩⎨⎧​0,m,m+n2nm​​if m=0 and n=0if n=0otherwise​

如果预计不需要调用并且也没有调用就直接给 1 的打分；一组内只有一条轨迹退化为 PPO 的判定方式，f(m,n)f(m,n)f(m,n) 即调和平均数，看 n、m 的相差程度。

# ARPO

熵：模型在某个位置预测下一个 token 时概率分布，代表模型预测判断是笃定还是分散不确定：

Ht=−∑v∈Vpt(v)log⁡pt(v)H_t = -\sum_{v \in V} p_t(v)\, \log p_t(v)Ht​=−v∈V∑​pt​(v)logpt​(v)

每次 tool call 后熵急剧增加（模型生成 token 不确定性增加，持续 10-15 个 token 后回落），那么在这个熵增加点像树一样分叉出多条采样，也就是在模型困惑的地方多次采样，最后的 RL 效果是否能更好。

ΔH1=Normalize(H1−Hinitial)\Delta H_1 = \text{Normalize}(H_1 - H_{initial})ΔH1​=Normalize(H1​−Hinitial​)：每次 tool call 后的约 10 多个 token 计算平均熵然后归一化，但 initial 是序列开头。PtP_{t}Pt​ 超过阈值 τ\tauτ 则进行分支操作。

最后的反向传播更新 loss 的时候对于这些 shared 部分就看作单独的一次 rollout，例如重复部分是 S1，然后独立的是 I1 和 I2，那么 S1+I1 和 S1+I2 看作两次不同的 rollout，选取的优势值也是 A1 和 A2 各自。

🤔 思考：类似蒙特卡洛搜索树，但不同的是外部的搜索和直接强化模型内部能力的不同。

# AEPO

ARPO 的变体，针对两个问题：

  1. 熵扎堆导致预算被少数轨迹垄断、好路径没被探索（rollout 坍缩）
  2. 探到的高熵 token 又被 clip 裁成零梯度

① Entropy Monitoring Pre-exploration（熵预监控）

  • HrootH_{root}Hroot​：问题本身的不确定性，模型读完问题、开始自己推理、但还没碰任何工具时，这段初始生成 token 的熵。
  • Htool_avgH_{tool\_avg}Htool_avg​：工具调用带来的平均不确定性（各 Tool-call Step 后 Htool1,Htool2,Htool3H_{tool1}, H_{tool2}, H_{tool3}Htool1​,Htool2​,Htool3​ 的平均）。

M=σ(β(Hroot−Htoolavg))M = \sigma\big(\beta (H_{root} - H_{tool_{avg}})\big)M=σ(β(Hroot​−Htoolavg​​))

M 大则多给全局轨迹采样，如果 M 小则多给局部探索。

② Entropy-Guided Balanced Beaming（熵引导的平衡分支）

Pt=(α+β⋅ΔHt)(1−P^(l))P_t = (\alpha + \beta \cdot \Delta H_t)(1 - \hat{P}(l))Pt​=(α+β⋅ΔHt​)(1−P^(l))

修改了 PtP_tPt​ 计算公式，当这条分支越多，P^(l)\hat{P}(l)P^(l) 越大，对应的 PtP_tPt​ 越小，更难开启分支。

针对问题二，在高熵 clip 项里加 stop-gradient：

Laepo=sg(Lclip)+α⋅(Lraw−sg(Lraw))L_{aepo} = \text{sg}(L_{clip}) + \alpha \cdot \big(L_{raw} - \text{sg}(L_{raw})\big)Laepo​=sg(Lclip​)+α⋅(Lraw​−sg(Lraw​))

Lclip=min⁡(rtAt,clip(rt,0.8,1.2)At)L_{clip} = \min(r_t A_t, \text{clip}(r_t, 0.8, 1.2) A_t)Lclip​=min(rt​At​,clip(rt​,0.8,1.2)At​)，由于超上限所以进行裁剪。未裁剪的原始项为 Lraw=rtAt=2.5L_{raw} = r_t A_t = 2.5Lraw​=rt​At​=2.5，在前向计算数值时，sg(x)\text{sg}(x)sg(x) 在数值上完全等于 xxx，所以 Laepo=LclipL_{aepo} = L_{clip}Laepo​=Lclip​。

但是在反向传播求导时：

∂Laepo∂rt=∂∂rt[sg(Lclip)]+α⋅∂∂rt[Lraw−sg(Lraw)]\frac{\partial L_{aepo}}{\partial r_t} = \frac{\partial}{\partial r_t} \Big[ \text{sg}(L_{clip}) \Big] + \alpha \cdot \frac{\partial}{\partial r_t} \Big[ L_{raw} - \text{sg}(L_{raw}) \Big]∂rt​∂Laepo​​=∂rt​∂​[sg(Lclip​)]+α⋅∂rt​∂​[Lraw​−sg(Lraw​)]

# GiGPO

Link

下面这幅图最左边是传统的 GRPO，中间是类似上面 AEPO 从当前节点开出新分支但算力消耗巨大，最右侧是该算法匹配轨迹中的相似 state（判定相似可能是余弦相似度等方法），然后再在组内计算优势和原先整个轨迹优势加权。

在对齐训练过程中优势 Advantage 计算时是根据整个 trajectory，但这就引发了稀疏奖励问题，也就是一次 step 的 reward 值可能需要看对之后步骤的影响。

AGiGPO=AE+w⋅ASA^{GiGPO} = A^E + w \cdot A^SAGiGPO=AE+w⋅AS

AEA^EAE 同 GRPO 不变，但是 ASA^SAS 会采用重新拉一个状态相同的组，根据原先 trajectory 中长远折扣回报，折扣因子 γ\gammaγ（例如：计算 a1(1)a_1^{(1)}a1(1)​ 在 step-level group 中的 reward 值的时候 G1(1)=0.95⋅r2(1)+0.952⋅r3(2)+0.953⋅r4(1)G_1^{(1)} = 0.95 \cdot r_2^{(1)} + 0.95^2 \cdot r_3^{(2)} + 0.95^3 \cdot r_4^{(1)}G1(1)​=0.95⋅r2(1)​+0.952⋅r3(2)​+0.953⋅r4(1)​）。

解决稀疏奖励问题，并且没有造成巨大开销，对于那些中间步骤 reward 得分 0 但对之后轨迹有重要作用的 step 进行二次奖励。

🤔 思考：变体 GAGPO 就是针对 GiGPO 的。

# Tree-GRPO

Link

🤔 思考：感觉这个和 ARPO 都是借鉴了蒙特卡洛搜索树那种结构，将外部搜索能力直接转化为模型内部的能力。

依旧是针对奖励稀疏、rollout 消耗高的问题：

  • Init：先让模型生成 MMM 条完整的 Chain，每条 Chain 的物理结构直接作为 MMM 棵初始树的骨架。
  • Sample：在每棵树中，随机选择 NNN 个非叶子节点（即不选最终答案节点，选中间的步骤）。
  • Expand：从这些被选中的中间节点出发，让模型继续往下生成新的后续轨迹（Branch），直到给出最终答案、遇到 <eos> 或达到最大工具调用步数（Action/Tool Budget）。
  • Loop：重复这个过程 LLL 轮。最终，一个 Prompt（问题）会得到一组极其丰富的树状 Rollout 结构。
